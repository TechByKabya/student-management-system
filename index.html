The Problem: Manual Attendance is Failing Us

At DIU, academic attendance relies on a system that is fundamentally slow, prone to human error, and heavily dependent on continuous server uptime.

1. The Time Cost of Calling IDs

In a standard classroom of 60 students, a teacher must open the portal, call out individual Student IDs, and manually click "Present" or "Absent". This archaic process consumes 10 to 15 minutes of valuable lecture time every single session.

2. "DIU Server is Down"

Teachers frequently encounter the frustrating reality of DIU server downtime or localized Wi-Fi outages. When the portal won't load, attendance must be taken on paper and entered later—leading to double data entry and frequent loss of records.

3. Proxy Attendance

In large lecture halls, students easily respond for absent friends, inflating attendance metrics and compromising academic integrity.

The Required Solution

We need a solution that is instantaneous, immune to network failures, fraud-proof, and cheap enough to deploy in every single DIU classroom without requiring a massive overhaul of the existing university database.

The Solution: Smart Edge Node

We engineered a dedicated ESP32-based hardware node that sits on the teacher's desk. Students simply tap their institutional ID card on the device as they enter. Attendance is logged instantly, accompanied by a confirming beep and green LED flash.

Real-world testing: Scanning student IDs autonomously during class entry.

Instantaneous Verification

By utilizing an RFID interface (PN532), scanning a student ID takes less than 0.5 seconds. A 60-student class is processed in under a minute as students walk through the door. Zero lecture time is lost.

Automated Parent SMS

The system automatically tracks missing students. The moment the teacher closes the session, SMS alerts are immediately dispatched to the parents of absent students, keeping them informed instantly.

Edge Computing

By processing the AI fraud detection directly on the ESP32 (Edge Compute), we avoid sending constant data streams to the cloud. This saves immense bandwidth and ensures decisions are made in under 10 milliseconds.

Asynchronous Processing

The hardware runs on FreeRTOS with dual-core processing. Core 0 handles all Wi-Fi network requests, while Core 1 handles biometric scanning. This prevents the scanner from "freezing" when waiting for the server to respond.

Breakthrough Innovation:
On-Device TinyML

How do we stop a student from bringing three friends' ID cards and tapping them all? Asking everyone to use a fingerprint scanner is too slow. Our core technological breakthrough is Tiny Machine Learning (TinyML).

The TinyML Neural Network

We did not just build a simple RFID scanner. We successfully deployed a trained, quantized Neural Network directly onto the ESP32's limited microcontroller memory. The TinyML algorithm acts as an invisible security guard.

Every time a card is tapped, the AI analyzes the temporal tapping behavior—measuring millisecond delays between scans, card frequency, and sequence patterns. It calculates a real-time Fraud Probability Score locally, without ever communicating with a cloud server.

Selective Biometric Challenge

95% of students scan normally and are approved instantly via RFID. But if the TinyML algorithm detects an anomaly (e.g., someone rapid-tapping three cards in a row), the device immediately halts. The OLED flashes "Fraud Alert", forcing the suspected student to authenticate via the built-in R307 fingerprint sensor. It is the perfect balance of extreme speed and bulletproof security.

100% Offline Resilience

What happens when the DIU server crashes or the classroom Wi-Fi goes down? With the current teacher portal, the system breaks. Our device doesn't.

SD-Card Cache & Hardware Clock

The device is equipped with a MicroSD card module and a battery-backed Real Time Clock (RTC). If the device loses connection to the university network, it switches to Offline Mode automatically. All attendance scans are cached securely on the SD card with perfect UTC timestamps. When the network returns, it batch-syncs the data to the server seamlessly. Zero data is ever lost.

Seamless API Integration

DIU does not need to throw away its current database or rewrite its servers to use our device.

Universal JSON Compatibility

The Smart Edge Node communicates using standard HTTP REST APIs and JSON payloads. Whether DIU’s backend is written in PHP, Node.js, Python, or Java, our device simply POSTs a JSON packet containing { "student_id": "252-15-254", "timestamp": "09:00:00" }. It is a completely plug-and-play edge computing layer.
